Chapter 6: Programming Languages

(Through page 340)

Definitions
machine language programs for modern computers consist of sequences of instructions that are encoded as numeric digits. Such an encoding system is known as a machine language
debugging Unfortunately, writing programs in a machine language is a tedious task that often leads to errors that must be located and corrected (a process known as debugging) before the job is finished.
assembler programs called assemblers were developed to convert mnemonic expressions into machine language instructions.
assembly languate A mnemonic system for representing programs is collectively called an assembly language.
trasnlator Once this collection of high-level primitives had been identified, a program, called a translator, was written that translated programs expressed in these high-level primitives into machine-language programs.
compiler Such a translator was similar to the second-generation assemblers, except that it often had to compile several machine instructions into short sequences to simulate the activity requested by a single high-level primitive. Thus, these translation programs were often called compilers.
interperter These programs were similar to translators except that they executed the instructions as they were translated instead of recording the translated version for future use.
cross-platform software That is, cross-platform software is software that is independent of an operating system’s design as well as the machine’s hardware design and is therefore executable throughout a network.
natural language a language that evolved over time without formal grammatical analysis
formal language a language that is precicely defined by grammers
machine independence programs that do not rely on the characteristics of a particular machine
high-level programming language High-level programming languages provide more abstractions for the programmer and make it easier for people to read and write a program
object oriented programming (OOP) a software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects. Together, these objects interact to solve the problem at hand.
scripting language These languages are typically used to perform administrative tasks rather than to develop complex programs.
data types:
  • Integer
  • float
  • string
  • numeric data consisting of whole numbers
  • numeric data that might contain values other than whole numbers
  • data consisting of several characters strung together
primitive data type The data types that are included as primitives in a programming language, such as int for integer and char for character, are called primitive data types.
data structure the conceptual shape or arrangement of data.
Python collection data structures
  • list
  • tuple
  • set
  • dictonary
  • a collection which is ordered and changeable. Allows duplicate members.
  • a collection which is ordered and unchangeable. Allows duplicate members.
  • a collection which is unordered, unchangeable (items are unchangeable however items can be added or removed), and unindexed. No duplicate members.
  • a collection which is ordered and changeable. No duplicate members. (no longer ordered as of version 3.7)
function a set of instructions for performing a task that can be used as an abstract tool by other program units.
local variable a variable that can only be refrenced within one function
global variable a variable whose scopes are not restricted to a aprticular part of a program
source program a program in its original form
object program the translated version of a program
